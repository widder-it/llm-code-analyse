Modul content — Fachliche Beschreibung

Zentrales Content-Management-Framework: strukturierte Inhalte, Rendering, CMS, Umfragen, FTP-Transfer und Volltextsuche.

16 Submodule 28 Klassen FreeMarker + FOP

Das content-Modul ist das Content-Management-Framework des Systems. Es verwaltet strukturierte Inhaltsobjekte (Content, DataResource) mit Assoziationen, Lokalisierung, Zugriffskontrolle und verschiedenen Rendering-Mechanismen. Es dient als gemeinsame Infrastruktur für alle anderen Module, die Inhalte anzeigen oder verwalten — Produktbeschreibungen, Profilbilder, Auftragsnotizen, CMS-Seiten, Blogs.

Spezialisierte Submodule erweitern den Kern: Umfragen mit PDF-Integration, FTP/SFTP-Dateitransfer, Blog-RSS-Feeds, CMS-Seitenverwaltung und PDF-Ausgabe via Apache FOP.

Abb. 1 — Zweistufiges Content-Modell: Struktur (Content) und Daten (DataResource)

Das Modell trennt bewusst Strukturknoten (Content) von den tatsächlichen Daten (DataResource). Ein Content-Knoten beschreibt Metadaten, Beziehungen und Zugriff; die DataResource enthält den eigentlichen Inhalt — als Text in der Datenbank, als Binärbild, als Dateipfad oder als URL.

ContentWorker ist die zentrale Rendering-Engine. Sie traversiert Content-Assoziationsbäume, löst Sprachvarianten auf und rendert den Inhalt je nach Mime-Typ über unterschiedliche Ausgabekanäle.

RendererAusgabe
FreeMarkerHTML, Text, XML — dynamisch

DirektauslieferungBilder, Dateien, Downloads
RSSBlog-Feeds (via BlogRssServices)
ContentMapFacade stellt Content-Daten als lazy-geladene Map für FreeMarker-Templates bereit — Content-Felder werden erst beim Zugriff aus der DB geladen.

3. CMS & Website-Verwaltung

Das CMS-Submodul verwaltet Webseiten als Content-Baum: Seiten bestehen aus verschachtelten Content-Knoten (Layouts, Blöcke, Texte, Bilder). Redakteure können Seiten anlegen, bearbeiten und veröffentlichen, ohne direkten Datenbankzugriff.

KonzeptBeschreibung
SubSiteEigene Website mit eigenem URL-Präfix
WebPageEine Seite im Content-Baum
LayoutWiederverwendbares Seitenlayout
BlockInhaltsbaustein (Text, Bild, HTML)
PUBLISH_LINKAssoziation: Inhalt ist veröffentlicht

Abb. 3 — CMS-Seitenstruktur: Website → Seite → Blöcke → Daten
URL-Auflösung: ContentUrlFilter löst alternative Content-URLs (Suffix -content) auf die echte Content-ID auf — ermöglicht sprechende URLs ohne eigene URL-Rewrite-Konfiguration.

4. Berechtigungen

ContentPermissionServices implementiert rollenbasierte Berechtigungsprüfungen für alle Content-Operationen. Geprüft wird gegen Content-Rollen, Party-Beziehungen und explizite Berechtigungseinträge — mit konfigurierbarer Fallback-Logik.

OperationPrüfung
LesenÖffentlich oder explizite Lesefreigabe
BearbeitenContent-Rolle AUTHOR oder EDITOR
VeröffentlichenRolle PUBLISHER
LöschenRolle OWNER oder Administratorrecht
Debug-Modus: PermissionRecorder protokolliert jeden Berechtigungsprüfungsschritt detailliert — nützlich bei komplexen Rollenhierarchien. Im Produktionsbetrieb deaktivieren.

5. Umfragen & PDF-Formulare

Das Survey-Submodul ermöglicht die Erstellung und Auswertung von Umfragen. Fragebögen werden über FreeMarker-Templates gerendert; Antworten werden je Umfrage und Befragten gespeichert und können ausgewertet werden.

FragetypBeschreibung
FreitextOffene Texteingabe
Single-ChoiceEine Antwort aus vorgegebenen Optionen
Multi-ChoiceMehrere Antworten möglich
SkalaBewertungsskala (1–5, 1–10)
DatumDatumseingabe mit Validierung

6. FTP / SFTP-Dateitransfer

FtpServices überträgt Content-Daten an externe Server über FTP oder SFTP. Die Zieladresse wird als FtpAddress-ContactMech konfiguriert und kann einer Party oder einem System zugeordnet werden.

ClientProtokoll
SimpleFtpClientStandard FTP (Apache Commons Net)
SshFtpClientSFTP über SSH (Apache MINA SSHD)
SecureFtpClientFTPS (Platzhalter, nicht implementiert)
FTPS nicht implementiert: SecureFtpClient enthält nur leere Methoden — FTPS (FTP über TLS) ist laut Kommentar im Code bewusst nicht realisiert.

7. Volltextsuche

ContentSearch implementiert eine strukturierte Volltextsuche über alle Content-Objekte. ContentKeywordIndex indiziert Keywords aus Content-ID, Name, Beschreibung und dem eigentlichen Inhalt. Der Suchzustand wird session-persistent von ContentSearchSession verwaltet.

<
SuchkriteriumBeispiel
VolltextStichwort im Inhalt oder Metadaten
Content-TypNur CMS-Seiten / Nur Bilder
StatusNur veröffentlichte Inhalte
SpracheNur Inhalte in de_DE
DatumZuletzt geändert in Zeitraum
Blog & RSS: BlogRssServices erzeugt RSS-Feeds aus veröffentlichten Blog-Artikeln (Content-Knoten mit PUBLISH_LINK- Assoziation und Status CTN_PUBLISHED). Feeds können von externen Readern und Aggregatoren abonniert werden.
