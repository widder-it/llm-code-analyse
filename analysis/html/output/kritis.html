KRITIS-Sicherheitsbewertung

Bewertungsstufe: Höchste KRITIS-Stufe · Analyse: 2026-04-15 · Grundlage: IT-SiG 2.0, NIS2, BSI TR-02102, BSI TR-03183

6 CRITICAL — Sofortmaßnahme 8 HIGH — Kurzfristig 6 Strukturelle Risiken
Akuter Handlungsbedarf für KRITIS-Betrieb. Das System weist 6 kritische Schwachstellen auf, die bei einem Betreiber der höchsten KRITIS-Stufe einen sofortigen Betriebsstopp erfordern würden — darunter Remote Code Execution auf der Laufzeitumgebung sowie mehrere XML External Entity (XXE)-Lücken. Hinzu kommen 8 Schwachstellen mit hohem Risiko sowie 6 strukturelle Sicherheitsrisiken, die einer umfassenderen Sanierung bedürfen.

1. CRITICAL — Sofortmaßnahmen

Diese Schwachstellen müssen ohne Wartungsfenster behoben werden. Jede der folgenden CVEs stellt für einen KRITIS-Betreiber ein K.O.-Kriterium dar.

3. Strukturelle Sicherheitsrisiken

Diese Risiken sind nicht durch einen einzelnen Patch behebbar — sie erfordern Architekturentscheidungen und mittelfristige Maßnahmen.

Kritisch
Legacy-Kryptographie (EntityCrypto)

Verwendete Algorithmen und Schlüssellängen in EntityCrypto sind aus dem kompilierten Code nicht vollständig rekonstruierbar. Schwache oder veraltete Verschlüsselung (DES, 3DES, RSA<2048) wäre für KRITIS nicht zulässig.

BSI TR-02102 Vollständige Kryptoanalyse der EntityCrypto-Implementierung; Abgleich gegen BSI TR-02102
Hoch
OWASP ESAPI — isValidSafeHTML deprecated

Die Methode isValidSafeHTML ist vom Hersteller für veraltet erklärt, weil sie XSS-Payloads als sicher einstufen kann. Überall wo sie zur Eingabevalidierung eingesetzt wird, besteht ein direktes XSS-Risiko.

OWASP ASVS V5 Alle Verwendungsstellen auditieren und durch sichere Alternative ersetzen
Hoch
~30 % Groovy-Code nicht vollständig analysierbar

Ca. 30 % der Klassen sind in Groovy implementiert. Die Dekompilierung liefert nur eingeschränkte Ergebnisse — Sicherheitsrelevante Logik in diesen Klassen kann nicht vollständig bewertet werden (blinde Flecken).

BSI IT-Grundschutz OPS.1.1.3 Groovy-Quellcode beschaffen oder durch unabhängige Methode analysieren (AST-Analyse, manuelle Prüfung)
Hoch
Supply-Chain-Angriffsfläche: 191 externe Abhängigkeiten

14 von 79 analysierten Libraries haben bekannte Schwachstellen. Die Gesamtzahl transitiver Dependencies ist deutlich höher. Kein automatisiertes CVE-Monitoring erkennbar.

IT-SiG 2.0 §8a / BSI TR-03183 Kontinuierliches SCA-Monitoring etablieren (z.B. OWASP Dependency-Check, Trivy); SBOM nach BSI TR-03183 führen
Mittel
8 eingebettete Payment-Gateway-Integrationen

Jede Gateway-Integration öffnet eine externe Kommunikationsverbindung. TLS-Pinning, Circuit Breaker und Anomalie-Erkennung sind im Code nicht erkennbar. Ausfall oder Kompromittierung eines Gateways kann den gesamten Zahlungsprozess betreffen.

PCI DSS v4 / BSI IT-Grundschutz NET.1 Alle externen Verbindungen dokumentieren, TLS-Konfiguration prüfen, Timeouts und Fallback-Strategien implementieren
Mittel
Keine erkennbare Netzwerk-Schichtentrennung im Code

Business-Logik, Datenzugriff und HTTP-Handler sind direkt gekoppelt (keine dedizierte Controller-Schicht). Eine Kompromittierung eines Moduls kann sich horizontal ausbreiten.

BSI IT-Grundschutz NET.3.2 Netzwerksegmentierung auf Infrastrukturebene kompensieren; Web Application Firewall vorschalten

4. Maßnahmenplan

Sofort Notfall-Patches — ohne Wartungsfenster
  • Apache Tomcat patchen (RCE CVE-2025-24813 + CLIENT_CERT-Bypass CVE-2026-29145)
  • Apache Tika patchen oder Datei-Upload bis zum Patch deaktivieren
  • iText ablösen oder XML External Entity Processing deaktivieren
  • JDOM patchen und setExpandEntities(false) als Sofortmaßnahme setzen
  • OWASP HTML Sanitizer patchen
Kurzfristig Innerhalb von 4 Wochen
  • Verbleibende Tomcat-CVEs patchen (HTTP Smuggling, OCSP, Token-in-Log)
  • Commons FileUpload auf 1.6 aktualisieren
  • Apache CXF auf 4.0.6 aktualisieren
  • XStream auf 1.4.21 aktualisieren
  • Log4j Socket Appender TLS-Konfiguration prüfen und patchen
  • Alle Logdateien auf versehentlich geschriebene Credentials / Tokens prüfen
Mittelfristig Innerhalb von 3 Monaten
  • EntityCrypto vollständig analysieren und gegen BSI TR-02102 prüfen
  • ESAPI isValidSafeHTML-Nutzung auditieren und ersetzen
  • Groovy-Quellcode beschaffen oder unabhängig analysieren
  • SCA-Monitoring automatisieren (OWASP Dependency-Check / Trivy in CI/CD)
  • SBOM nach BSI TR-03183 erstellen und pflegen
  • Penetrationstest mit Fokus XXE, HTTP Smuggling, Auth-Bypass beauftragen
Strukturell Dauerhaft / laufend
  • Meldeprozess nach IT-SiG 2.0 §8b sicherstellen (24h-Meldepflicht bei Vorfällen)
  • Web Application Firewall zum Schutz vor HTTP Smuggling und XXE vorschalten
  • Alle externen Gateway-Verbindungen dokumentieren und TLS-Konfiguration überwachen
  • PCI DSS v4 Compliance für Payment-Gateway-Integrationen sicherstellen
  • BSI IT-Grundschutz Assessment beauftragen
  • Netzwerksegmentierung auf Infrastrukturebene prüfen und ggf. verstärken

5. Relevante Rechtsgrundlagen & Normen

Norm / Gesetz Titel Relevanz für dieses System
IT-SiG 2.0 §8a Technische und organisatorische Maßnahmen Betreiber kritischer Infrastrukturen müssen angemessene Vorkehrungen treffen. Offene CRITICAL-CVEs in Produktivsystemen verletzen diese Anforderung.
IT-SiG 2.0 §8b Meldepflicht Erhebliche Störungen sind dem BSI innerhalb von 24 Stunden zu melden. RCE oder Auth-Bypass-Ausnutzung sind meldepflichtige Sicherheitsvorfälle.
BSI TR-02102 Kryptographische Verfahren Die ungeklärte EntityCrypto-Implementierung muss gegen die BSI-Empfehlungen zu Algorithmen und Schlüssellängen geprüft werden.
BSI TR-03183 Software Bill of Materials (SBOM) KRITIS-Betreiber sind verpflichtet, eine aktuelle SBOM zu führen und bekannte Schwachstellen kontinuierlich zu überwachen.
NIS2-Richtlinie EU-Netzwerk- und Informationssicherheit Supply-Chain-Sicherheit, Patch-Management und Incident Response sind explizite NIS2-Anforderungen für wesentliche Einrichtungen.
PCI DSS v4 Payment Card Industry Data Security Standard Die 8 eingebetteten Payment-Gateway-Integrationen unterliegen PCI DSS. TLS-Konfiguration, Logging und Zugangskontrolle der Gateways müssen geprüft werden.
